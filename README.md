SortingAlgorithms is a C++ console application that implements several classic sorting algorithms and measures their execution time across multiple container types. The project is designed as a comparative learning tool for studying algorithm behavior, relative performance, and Big O characteristics using the same data-generation and timing workflow.

## Key Features

- Implements multiple sorting algorithms in separate header-based classes:
  - Insertion Sort
  - Bubble Sort
  - Improved Bubble Sort
  - Heap Sort
  - Selection Sort
  - Shell Sort
  - Merge Sort
  - Quick Sort
- Uses a shared `ISort<T>` interface for consistent naming, container access, and timing-oriented execution.
- Benchmarks algorithms against three container styles:
  - `std::vector<int>`
  - raw C-style arrays
  - a custom `Array<T>` container
- Generates fresh random integer datasets for each benchmark run.
- Writes timing results to `sorting_results.txt`.
- Includes a custom `Array<T>` implementation and custom exception type for array-related operations.
- Organizes the project as a Visual Studio C++ solution.

## Architecture Overview

This repository is structured as a single C++ console application with no separate frontend or database layer. The executable entry point in `Main.cpp` orchestrates a series of benchmark runs by invoking helper functions that prepare data, execute each sorting implementation, and append timing output to `sorting_results.txt`.

At the core of the design is the templated `ISort<T>` base class, which acts as a common contract for all sort implementations. Each algorithm class inherits from `ISort<T>`, receives a pointer to a container plus its size, and implements a `Sort()` method that performs the algorithm and returns elapsed time in milliseconds. This gives the project a lightweight polymorphic structure centered around a shared interface.

The benchmarking workflow is coordinated through `data_fill.hpp`, which:
- creates randomized datasets,
- refreshes container contents before each sort,
- runs the algorithms against vectors, C-style arrays, and the custom `Array<T>` type,
- records timing output to a text file.

Notable visible patterns in the codebase include:
- **Template-based generic programming** for reusable sorting implementations
- **Interface-style abstraction** through `ISort<T>`
- **Separation of concerns** between algorithm implementations, dataset preparation, and application entry point

## Getting Started

### Prerequisites

- Windows
- Visual Studio with C++ desktop development tools
- MSVC toolset compatible with the project file (`v142` appears in the project configuration)

### Installation

```bash
git clone https://github.com/drussell33/SortingAlgorithms.git
cd SortingAlgorithms
```

### Usage

#### Option 1: Run with Visual Studio

1. Open `SortingAlgorithms.sln` in Visual Studio.
2. Build the solution.
3. Run the project.
4. Review the generated output in `SortingAlgorithms/sorting_results.txt` or the working directory output file.

#### Option 2: Build from the Developer Command Prompt

```bash
msbuild SortingAlgorithms.sln /p:Configuration=Release
```

Then run the compiled executable from the generated output directory in Visual Studio's build output.
